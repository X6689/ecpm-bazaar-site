# eCPM Bazaar Outreach Platforms 调研笔记

调研日期：2026-06-24
调研目标：寻找适合 eCPM Bazaar 发布内容、评论互动、验证需求的海外游戏开发者平台。

## 关键问题
1. 哪些平台聚集了独立游戏、手游、App 变现相关开发者？
2. 哪些平台适合新账号先评论/回复，而不是直接硬发产品链接？
3. 哪些平台适合提交产品或长期展示 eCPM Bazaar？

## 发现

### 游戏开发社区优先

- itch.io Community 有独立的 Community 页面，其中 Devlogs 板块说明开发者可以介绍 WIP、分享更新，任何人都可以评论 devlog。适合先评论开发者项目、后续再发一个“mobile ad revenue diagnosis checklist”的讨论帖。来源：https://itch.io/community、https://itch.io/board/10021/devlogs
- GameDev.net 已经验证可用：eCPM Bazaar 相关讨论帖已通过审核，并收到回复。适合继续低频跟进，不要重复发同主题。
- Unity Discussions 有移动游戏变现、AdMob、Unity Ads 等历史讨论。更适合在相关问题下提供诊断框架，不适合直接发产品推广帖。来源：https://discussions.unity.com/t/monetizing-for-mobile-suggestions/943869
- GameDev.tv Community 出现过“mobile game monetization with ads / Unity Ads / AdMob”相关讨论，说明有学习型开发者会问广告变现问题。适合评论帮助型答案。来源：https://community.gamedev.tv/t/mobile-games-which-is-the-better-choice-for-monetization-with-ads/254966
- B4X Forum 有 AdMob 收入相关讨论。B4X 面向 Android/iOS app 开发者，和“app ad revenue diagnosis”相关度较高。适合后续注册观察。来源：https://www.b4x.com/android/forum/threads/when-using-admob-what-are-the-chances-of-actually-making-money.149394/
- GameMaker Community 有 monetization 标签。适合找移动/HTML5/广告变现问题进行评论，不适合一上来开产品帖。来源：https://forum.gamemaker.io/index.php?tags/monetization/
- Google AdMob API Developer Forum / Google Groups 是技术支持场景，只适合 API/SDK/报表技术问题，不适合推广。可以用来找真实痛点关键词。来源：https://groups.google.com/g/google-admob-api-developer-forum、https://support.google.com/admob/answer/2740010?hl=en

### 产品目录 / 发布平台

- TinyLaunch 已提交并排期：2026-07-20，URL：https://tinylaunch.com/launch/16633
- DEV.to 已发多篇文章，建议 5-7 天一篇，不要日更硬广。
- Fazier / Uneed / BetaList / MicroLaunch 属于产品目录方向，适合做 SEO 和早期曝光；其中部分平台有付费提速或提交门槛，不作为每日互动主阵地。
- Product Hunt 适合产品更完整后做一次正式 launch。现在可先准备材料，不急着发布。

## 调研结论

### 当前推荐优先级

1. itch.io Community / Devlogs：最适合作为下一个新平台，先评论，再发讨论。
2. GameDev.net：继续维护已通过审核的帖子，等回复后再跟进。
3. GameDev.tv Community：找移动广告变现问题，发帮助型回复。
4. Unity Discussions：只在相关问题下回复，不开推广帖。
5. B4X Forum：适合 Android/App/AdMob 方向，后续可注册观察。
6. GameMaker Community：补充平台，找 monetization 相关帖。
7. Product directories：TinyLaunch、Fazier、Uneed、BetaList、MicroLaunch、Product Hunt，用于产品展示，不作为每日评论主阵地。

### 2026-06-26 补充候选平台

- GDevelop Forum: https://forum.gdevelop.io/  
  适合无代码/轻量游戏开发者，很多小团队和个人开发者。先找 showcase/devlog/feedback 帖评论，后续可发“small mobile game metrics checklist”讨论。
- Construct Community: https://www.construct.net/en/forum  
  Construct 用户常做 HTML5/mobile 游戏，适合找 monetization、mobile export、feedback 类帖子。
- Defold Forum: https://forum.defold.com/  
  Defold 面向轻量跨平台游戏，开发者质量较高。适合低频、高质量技术/产品反馈。
- Cocos Forum: https://forum.cocosengine.org/  
  Cocos 有较多 mobile game 开发者，和广告变现相关度可能比纯 PC 独游社区更高。
- Solar2D Forum: https://forums.solar2d.com/  
  Solar2D 常用于 2D mobile game，可能有 AdMob/Unity Ads/monetization 旧帖可回复或学习痛点。
- HTML5 Game Devs Forum: https://www.html5gamedevs.com/  
  适合浏览器/HTML5 游戏开发者，偏技术和引擎。可找 mobile/browser/game feedback 话题。
- Exilian Forum: https://exilian.co.uk/forum/  
  老式独立开发社区，适合小众、长期交流，但流量可能低。
- r/indiegamedevforum: https://www.reddit.com/r/indiegamedevforum/  
  Reddit 补充社区。仍按低风险策略：只评论游戏反馈，不带链接。
- Google Mobile Ads SDK Developers Group: https://groups.google.com/g/google-admob-ads-sdk  
  注意：搜索结果显示该论坛从 2026-01-19 起只读，不能发新帖/回复；只能作为历史痛点搜索来源。
