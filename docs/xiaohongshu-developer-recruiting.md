# 小红书开发者试点招募文案

## 一句话结论

可以在小红书找开发者，但不要上来就说“把 TopOn Publisher Key 给我”。更合适的表达是：免费帮小游戏/App 开发者做一次广告变现体检，确认有合作意愿后再用只读报表 Key 做本机接入。

## 推荐标题

- 小游戏广告收入突然掉了？我免费帮 3 个开发者看一次 eCPM
- 找 3 个有广告收入的独立开发者，免费做广告变现诊断
- 做小游戏/App 的朋友，你们每天看 eCPM 和填充率吗？

## 正文版本

最近在做一个广告变现数据工具，想找 3 个真实小游戏/App 开发者做免费试点。

适合你：

- 已经接了 TopOn / 聚合广告
- 每天有展示、收入或 eCPM 数据
- 想知道收入下跌是 eCPM、填充率、展示量还是国家流量变化导致的
- 不想每天手动翻广告后台

我可以免费帮你做：

- eCPM / 收入 / 展示量 / 填充率看板
- 最近 7-10 天异常波动诊断
- 广告位、国家、广告源维度拆解
- 收入下跌告警规则建议
- 一份可读的优化建议

安全说明：

- 不需要你的 TopOn 登录账号和密码
- 优先使用只读报表权限或子权限 Key
- Key 只用于读取报表数据，不做后台配置操作
- 不公开你的 App 名称、收入截图和具体数据
- 试点结束后可随时撤销授权

想参与的话可以私信我：

1. 你的产品类型：小游戏 / App / 小程序
2. 当前广告平台：TopOn / AdMob / AppLovin / 其他
3. 每天大概是否有广告展示数据
4. 你最想解决的问题：收入波动 / eCPM 低 / 填充率低 / 不知道怎么看数据

先做 3 个样本，免费。

## 私信回复模板

你好，我这边是做广告变现数据诊断工具的。为了保护你的账号安全，不需要你提供登录账号密码。

如果你愿意试点，推荐流程是：

1. 你先确认 TopOn 后台最近 7-10 天有报表数据。
2. 使用 Publisher Key 或子权限 Key 做只读报表接入。
3. 我只在本机/本项目中读取报表，不公开你的数据。
4. 跑完后给你一份 eCPM、收入、填充率、广告位/国家维度的诊断结果。

如果你担心安全，可以先打码发一张数据报表页截图，我判断这个工具能不能帮上忙。

## 避坑

- 不要承诺“保证收入提升”，只能承诺“发现问题和给优化建议”。
- 不要让对方发账号密码。
- 不要公开展示开发者真实收入截图，除非获得明确授权并打码。
- 不要诱导刷广告、点广告或制造虚假展示。
- 第一次试点尽量找有真实收入、愿意沟通的小团队或独立开发者。
